# Halo 瞬间发布

通过 AstrBot 将 QQ、微信等聊天平台中的文字和图片发布到 Halo 2.x「瞬间」插件。

## 前置条件

- Halo 2.x 已安装并启用「瞬间」插件。
- 在 Halo Console 中创建个人访问令牌（PAT），并授予瞬间管理、附件查看和附件管理权限。
- AstrBot 中发送指令的账号已被设为管理员；也可以在插件配置中调整权限。

## 配置

安装插件后，在 AstrBot WebUI 的插件配置页面填写：

- `Halo 站点地址`：例如 `https://blog.example.com`，不要包含 `/console`。
- `Halo 个人访问令牌`：填写 PAT，既可以带 `Bearer ` 前缀，也可以只填令牌。
- `附件存储策略名称`：填写 Halo Policy 的 `metadata.name`。如果 Halo 只有一个存储策略，可以留空让插件自动选择；存在多个策略时必须填写。
- 其余配置可按需设置，包括可见性、默认标签、附件分组、草稿超时、图片限制和用户权限。

如果 Halo 使用自签名 HTTPS 证书，可以关闭证书校验，但不建议在公网环境这样做。

## 推荐用法：草稿收集模式

QQ、微信经常会将文字和图片拆成多条消息。使用草稿模式可以分别发送：

```text
/halo开始 今天天气很好
```

接下来直接发送一条或多条文字、图片消息，机器人会逐条加入当前草稿。完成后发送：

```text
/halo发布
```

草稿相关指令：

- `/halo开始 [初始文字]`：创建新草稿。
- `/halo预览`：查看文字预览、图片数量和超时时间。
- `/halo发布`：上传图片并发布；失败时保留草稿，方便重试。
- `/halo取消`：取消草稿并清理暂存图片。

草稿按照“聊天会话 + 发送者”隔离，因此群聊中的不同用户不会互相串稿。默认 30 分钟无操作后自动清理，可在插件配置页面修改。

## 快速发布

如果消息平台支持把文字和图片放在同一条消息中，也可以直接发送：

```text
/halo 今天天气很好。
```

可以在同一条消息中附带图片，也可以只发送 `/halo` 加图片。图片会先上传到 Halo 附件库，再作为 `PHOTO` 类型写入瞬间。

## 常见问题

- `HTTP 400 / Invalid part of policyName`：填写有效的附件存储策略 `metadata.name`；不要填写展示名称。
- `HTTP 401`：PAT 无效或已过期。
- `HTTP 403`：PAT 对应用户缺少瞬间管理、附件管理或存储策略查看权限。
- 图片转换失败：确认 AstrBot 所使用的消息平台能够提供可访问的图片 URL 或本地缓存。
- 指令无人响应：确认 AstrBot 的唤醒前缀；默认指令以 `/` 开头。
